# Botium Toys Security Audit

## Project Overview
This project is a mock internal security audit for Botium Toys. The goal was to assess current controls, identify compliance gaps, and recommend improvements to strengthen the company’s security posture.

## Skills Demonstrated
- Security auditing
- Risk assessment
- Controls assessment
- PCI DSS, GDPR, and SOC compliance review
- NIST Cybersecurity Framework concepts
- Documentation and reporting

## Key Findings
- Lack of encryption for customer payment data
- No disaster recovery plan
- No backups of critical data
- No IDS installed
- Weak password policy
- No centralized password management system
- Lack of least privilege and separation of duties

## Recommendations
- Implement encryption for sensitive data
- Enforce least privilege access
- Create a disaster recovery plan
- Establish regular backups
- Deploy an IDS
- Strengthen password policies
- Implement a password management system
- Improve asset inventory and classification

## Outcome
This audit identified high-risk gaps and provided prioritized recommendations to reduce risk, improve compliance, and strengthen Botium Toys’ security posture.
